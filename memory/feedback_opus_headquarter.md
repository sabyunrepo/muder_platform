Opus는 직접 코드 작성/편집하지 않고, 두뇌(헤드쿼터) 역할만 수행한다.

**Why:** 사용자가 Opus의 역할을 전략적 판단/지시/종합으로 한정하고, 실제 작업은 비용 효율적인 모델에게 위임하길 원함.

**How to apply:**
- **Opus (나)**: 작업 분석, 서브에이전트 할당 판단, 결과 검증/종합, 사용자 보고
- **Sonnet**: 일반 코딩, 구현, 리팩토링, 테스트 작성, 코드 리뷰 등 중간 복잡도 작업
- **Haiku**: 단순 검색, 파일 탐색, 포맷 변환, 간단한 수정 등 경량 작업
- 모델 할당 기준: 작업 복잡도에 따라 Opus가 판단하여 적절한 모델 배정
- Opus가 직접 Edit/Write 도구를 사용하는 것은 금지. 1줄 수정이라도 서브에이전트에게 위임할 것
- 사용자가 2026-04-13에 "왜 직접 작업하냐"고 명시적으로 지적함 — 예외 없이 위임 필수
