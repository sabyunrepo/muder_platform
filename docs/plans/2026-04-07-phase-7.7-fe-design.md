# Phase 7.7 Step 3+4+5 — FE 미디어/오디오/리딩 설계

**작성일**: 2026-04-07
**상태**: 승인됨 (brainstorming 완료)
**범위**: Phase 7.7 프론트엔드 (에디터 미디어 탭 + 오디오 4채널 + 리딩 UI + 컷신/증거 비디오)
**선행**: Phase 7.7 Step 1 (media DB + R2 + editor service), Step 2 (게임 엔진 오디오 통합)

## 목표

Phase 7.7의 프론트엔드 완결. 제작자가 에디터에서 BGM/SFX/Voice/Video 미디어를 관리하고 리딩 섹션을 편집할 수 있게 하고, 게임 플레이 중 4채널 오디오 + 컷신/증거 영상 재생 + 줄 단위 제어가 가능한 리딩 오버레이를 제공한다.

## 설계 원칙

- 서버는 "무엇을 재생하라" 지시만 발행, 실제 재생은 100% 클라이언트
- Manager 레이어는 단일 책임 (BgmManager/VoiceManager/VideoPlayer 독립)
- Orchestrator가 크로스 관심사 조합 (리딩↔BGM, 컷신↔BGM)
- VideoPlayer 인터페이스 추상화로 Phase 7.8+ 파일 비디오 확장 대비
- 기존 에러 처리 체계(AppError/ErrorBoundary) 전면 재사용
- 프로젝트 규칙 준수: Tailwind CSS only, lucide-react, Zustand 3레이어

## 릴리스 단위

Step 3+4+5 **단일 PR** (브랜치: `feat/phase-7.7-fe`).

## 3. 데이터 플로우

### 3.1 파일 업로드

```
MediaUploadModal
  → POST /editor/themes/{id}/media/upload-url (RequestMediaUpload)
  → PUT presigned URL + onprogress (XHR)
  → POST /editor/themes/{id}/media/confirm
  → invalidate(['media', themeId])
  → MediaTab 목록 갱신
```

### 3.2 리딩 + 자동 보이스

```
엔진: phase_enter → ReadingModule.start
  → S→C reading:started { sectionId, lines[], advanceBy(per line), bgmMediaId }

useGameMediaEvents:
  ├─ readingStore.loadLines(lines)
  └─ orchestrator.handleSetBgm(bgmMediaId)

S→C reading:line_changed { index }
  ├─ readingStore.showLine(index)
  └─ 현재 줄 voiceMediaId 있으면:
      VoiceManager.enqueue(voice) → onEnded → C→S reading:voice_ended

advanceBy 분기:
  ├─ voice: 서버가 voice_ended 수신 → 다음 line_changed 발행
  ├─ gm:    GM "다음" → C→S reading:advance (서버 권한 검증)
  └─ role:<roleId>: 해당 역할 플레이어 "다음" → C→S reading:advance

플레이어 이탈:
  → presence 감지 → S→C reading:paused { playerId }
  → 모든 클라이언트 버튼 비활성화

재접속:
  → WS 핸드셰이크 → S→C reading:state { sectionId, currentIndex, lines }
  → readingStore.restore(state)

모든 필요 플레이어 복귀:
  → S→C reading:resumed → 진행 재개

완료:
  → S→C reading:completed → readingStore.clear()
```

### 3.3 컷신 비디오

```
엔진: ActionPlayMedia(mediaId, mode=cutscene, bgmBehavior=pause)
  → S→C media:play_video { mediaId, mode, skippable, bgmBehavior }

useGameMediaEvents:
  → videoOrchestrator.playCutscene(mediaId, opts)
      ├─ fetch media 메타데이터
      ├─ createVideoPlayer(media) (YouTubeVideoPlayer)
      ├─ bgmBehavior 처리: pause | keep | stop
      └─ CutsceneModal.open()

VideoPlayer.onEnded:
  → C→S media:video_ended
  → bgmBehavior=pause 시 BgmManager.resume()
  → CutsceneModal.close()

GM 스킵:
  → C→S media:video_skip → 서버가 전체 방에 video_ended 발행

재접속 중 컷신:
  → 처음부터 재생 (seek 복잡도 회피)
```

### 3.4 증거 영상 (B)

BGM/게임 흐름 건드리지 않음. EvidenceVideoCard가 독립 VideoPlayer 인스턴스 소유, 사용자 자유 재생. 서버 이벤트 불필요.

## 4. 에러 처리 + 엣지 케이스

### 4.1 기존 체계 재사용

- **백엔드**: `apperror.AppError` + `WrapHandler` + `WriteError` + Sentry/OTel 그대로
- **프론트엔드**: `ApiHttpError` + `showErrorToast` + 3계층 ErrorBoundary 그대로
- **에러 코드 형식**: `DOMAIN_REASON` (실제 codes.go 컨벤션)

### 4.2 신규 에러 코드

```go
// Reading
ErrReadingSectionNotFound  = "READING_SECTION_NOT_FOUND"
ErrReadingAdvanceForbidden = "READING_ADVANCE_FORBIDDEN"
ErrReadingInvalidAdvanceBy = "READING_INVALID_ADVANCE_BY"
```

FE 매퍼에 한국어 메시지 + params 치환 추가.

### 4.3 엣지 케이스 처리

| 시나리오 | 처리 |
|---|---|
| 업로드: presigned URL 만료 | 재시도 버튼 → URL 재요청 |
| 업로드: R2 PUT 실패 | 3회 재시도 (1/2/4s), 실패 시 uploadId 폐기 |
| 업로드: 파일 크기 초과 | 클라이언트 사전 차단 |
| 업로드: 쿼터 초과 | 서버 4xx + 토스트 |
| YouTube: 비공개/삭제 | oEmbed 404 → 에러 메시지 |
| YouTube: IFrame 로드 실패 | 3회 재시도 → 자동 스킵 |
| BGM 크로스페이드 중첩 | 현재 페이드 즉시 완료 → 새 페이드 |
| Voice 중 라인 변경 | stopCurrent → 새 큐 |
| Voice 미디어 404 | voice_ended 즉시 발행 |
| Web Audio suspended | resume 시도, 실패 시 silent fail |
| reading:advance 권한 없음 | 서버 403 (UI는 본래 버튼 숨김) |
| role 플레이어 이탈 | **정지** (reading:paused), GM 폴백 없음 |
| 리딩 BGM 삭제됨 | 경고 로그, BGM 없이 진행 |
| voice 모드인데 voice 없음 | 에디터 저장 시점에 거부 |
| WS 재연결 | `reading:state` 재전송, 현재 인덱스부터 이어서 |
| 컷신 중 재접속 | **처음부터 재생** (seek 복잡도 회피) |
| 컷신 중첩 요청 | VideoOrchestrator singleton, 무시 + 경고 |
| bgmBehavior=pause 실패 | finally 블록에서 resume 보장 |
| 미디어 삭제 시 참조 있음 | 409 + 참조 목록 모달 |
| 에디터 동시 편집 | optimistic locking, 충돌 시 새로고침 유도 |
| 메타데이터 XSS | 입력 검증 + 렌더 시 escape |

### 4.4 권한/보안

- 미디어 CRUD: 테마 소유자/협업자만
- 다른 테마 미디어 접근 차단 (404 마스킹)
- presigned URL 5분 TTL + 로그 미노출
- `advanceBy` 권한 검증은 서버 전용
