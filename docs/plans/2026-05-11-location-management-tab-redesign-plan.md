# 장소 관리 탭 UI/UX 재설계 실행 계획

## 0. 이번 설계의 고정 결정

- `맵`은 조사 계층이 아니라 장소를 묶는 관리/배치 컨테이너다.
- 장소는 모두 동등한 조사 단위다. 하위장소/부모장소를 만들지 않는다.
- 조사 계층은 `맵 → 장소 → 단서`로 고정한다.
- `리스트형`은 장소에 배치된 단서를 목록으로 확인하고 획득하는 방식이다.
- `카드덱형`은 장소에 단서를 추가하면 그 단서가 자동으로 덱 카드가 된다.
- 카드덱형 MVP에서는 별도 `슬롯 추가/슬롯 관리` UI를 만들지 않는다.
- 단서는 전역 유니크다. 한 번 배치된 단서는 다른 장소/하위항목/덱에 중복 배치할 수 없다.
- 이번 범위는 에디터 UI/UX와 저장 계약 정리다. 플레이 런타임의 실제 단서 소모, 동시 클릭 충돌, 재접속 복원은 후속 runtime engine 작업으로 분리한다.

## 1. 목표

제작자가 장소 탭에서 다음 질문에 바로 답할 수 있게 만든다.

- 이 장소는 어느 장면부터 보이고 언제 숨겨지는가?
- 이 장소는 모든 캐릭터가 조사할 수 있는가?
- 리스트형이면 이 장소에 어떤 단서가 목록으로 배치되는가?
- 카드덱형이면 이 장소에서 어떤 단서 카드가 순차/랜덤으로 지급되는가?
- 현재 설정에서 빠진 단서와 빠진 공개 시점은 무엇인가?

- [ ] 검수 메시지는 보이지만 바로 수정할 섹션으로 연결되는 구조가 약하다.

## 4. UI 규칙

### 맵/장소 트리

- [ ] `맵`은 트리 루트가 아니라 별도 선택 영역으로 표시한다.
- [ ] 장소 트리에는 장소와 하위 조사 항목만 표시한다.
- [ ] 상위 장소에는 `+ 하위 조사 항목 추가`를 보여준다.
- [ ] 하위 조사 항목에는 `+ 하위 조사 항목 추가`를 보여주지 않는다.
- [ ] 장소와 하위 조사 항목을 시각적으로 구분한다.
  - 장소: 큰 구역 아이콘/굵은 텍스트
  - 하위 조사 항목: 들여쓰기/작은 아이콘/`조사 항목` 뱃지
- [ ] 장소 검색은 장소와 하위 조사 항목을 모두 검색하되 결과에서 부모 경로를 보여준다.

### 탐색 구조 캔버스

- [ ] 리스트형일 때 중앙 캔버스 제목은 `하위 조사 항목`으로 표시한다.
- [ ] 리스트형은 `거실 자체를 조사하지 않고 아래 항목을 조사합니다`를 설명한다.
- [ ] 리스트형은 하위 항목별 단서 수를 표시한다.
- [ ] 카드덱형일 때 중앙 캔버스 제목은 `장소 덱 단서`로 표시한다.
- [ ] 카드덱형은 `이 장소에 배치된 단서가 플레이 중 한 장씩 지급됩니다`를 설명한다.
- [ ] 카드덱형은 별도 슬롯 추가 버튼을 만들지 않는다.
- [ ] 카드덱형에서 `+ 단서 추가`로 단서를 추가하면 즉시 덱 카드가 된다.
- [ ] `deckOrder=fixed`면 단서 순서를 번호로 표시한다.
- [ ] `deckOrder=random`이면 단서를 `랜덤 후보` 목록으로 표시한다.

### 설정 패널

- [ ] 선택한 장소 또는 하위 조사 항목의 설정만 보여준다.
- [ ] 섹션 순서는 `기본 정보 → 공개 시점 → 조사 구조 → 접근 제한 → 단서 배치 → 검수`로 둔다.
- [ ] 내부 필드명, raw JSON, config key는 노출하지 않는다.
- [ ] `출현 장면`이 비어 있으면 `처음부터 공개`로 설명한다.
- [ ] `숨김 장면`이 비어 있으면 `마지막 장면까지 공개`로 설명한다.
- [ ] 접근 제한은 `조사 불가 캐릭터` 문구를 중심으로 표현한다.
- [ ] 검수 항목은 클릭 시 관련 섹션으로 이동하거나 해당 섹션을 펼친다.

## 5. 데이터/검증 규칙

### 장소 계층

- [ ] 서버에서 부모 장소가 이미 부모를 가지고 있으면 새 자식을 만들 수 없게 막는다.
- [ ] 서버 에러 의미: `하위 조사 항목 아래에는 다시 하위 항목을 만들 수 없습니다.`
- [ ] UI에서도 하위 조사 항목 선택 시 자식 생성 액션을 숨긴다.
- [ ] 부모 장소가 숨겨지면 하위 조사 항목도 함께 숨겨진다는 설명을 검수/도움말에 노출한다.

### 리스트형 저장 의미

```ts
{
  locationId: 'living-room',
  subLocationId: 'drawer',
  clueId: 'receipt',
  order: 0
}
```

- [ ] `subLocationId`는 같은 맵의 2단계 하위 조사 항목이어야 한다.
- [ ] 상위 장소 자체를 `subLocationId`로 쓰지 않는다.
- [ ] 같은 `clueId`는 한 번만 배치 가능하다.

### 카드덱형 저장 의미

```ts
{
  locationId: 'living-room',
  clueId: 'receipt',
  order: 0
}
```

- [ ] 카드덱형에서는 `slotId`를 UI에 노출하지 않는다.
- [ ] 카드덱형에서는 `label`을 UI에 노출하지 않는다.
- [ ] 장소에 단서를 추가하면 그 단서가 덱 카드가 된다.
- [ ] `order`는 순차 모드의 획득 순서다.
- [ ] 랜덤 모드에서도 `order`는 편집 안정성을 위한 표시 순서로만 사용한다.
