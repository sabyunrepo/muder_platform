## HIGH (unique)

### H-1 [Security] Public repo fork PR cache poisoning (4 runner 횡전파)
- **Reviewer**: security
- **사실**: `gh repo view sabyunrepo/muder_platform` → `isPrivate=false` (PR description 의 "private repo" 전제 오류). `pull_request` 이벤트 + `runs-on: self-hosted` → fork PR 의 untrusted code 가 `GOMODCACHE`/`PNPM_STORE`/`Playwright cache` 에 악성 binary/패키지 변조본 주입 시 4 runner 의 다음 main/PR job 에 즉시 횡전파.
- **현재 보호**: `pull_policy: never` + 호스트 named volume → 한 번 오염되면 image rotation 으로 정화 불가, `docker volume rm` manual 만 복구.
- **권고**: (a) `if: github.event.pull_request.head.repo.fork == false` 게이트, (b) GitHub "Require approval for first-time contributors" 정책 활성, (c) fork PR 만 ephemeral non-shared cache 분리.

### H-2 [Performance + Test] `setup-go` / `setup-node` 자체 cache 와 충돌 (GHA cache miss)
- **Reviewer**: performance + test (중복 검출)
- **사실**: `actions/setup-go@v5` 가 `cache-dependency-path: apps/server/go.sum` 으로 `GOMODCACHE`/`GOCACHE` 기반 GHA cache restore 수행. PR-4 가 두 변수를 `/opt/cache/go/{mod,build}` 로 덮어쓰면 GHA 가 계산한 key 와 실제 복원 경로 불일치 → **GHA cache restore 영구 miss**. 동일 문제 `actions/setup-node` `cache: 'pnpm'` ↔ `PNPM_STORE_PATH` 변경.
- **결과**: PR-4 의 named volume cache 는 채워지지만 GH-managed cache (PR 별 무료) 가 무효화 → 실제 효과는 PR-4 가 의도한 것보다 작음. 또는 음의 효과 (GHA cache 무효 + volume cache miss 첫 run).

### H-3 [Test + Architecture] `ci.yml` 4 job 적용 범위 밖
- **Reviewer**: test + architecture (간접)
- **사실**: `ci.yml` 의 `runs-on: self-hosted` (no `containerized` 라벨) → PR-4 cache volume 영향 0. 적용은 `e2e-stubbed.yml` 2 job 뿐 (PR-167 30a263d 가 변경). PR description 의 "CI 시간 ~70% 단축" 범위와 실제 적용 불일치.
- **권고**: `ci.yml` 도 `[self-hosted, containerized]` 로 같이 옮기거나, PR description 의 효과 범위를 e2e-stubbed.yml 한정으로 정정.

### H-4 [Architecture] 격리 원칙 약화 (race-safe 주장 검증 부재)
- **Reviewer**: architecture
- **사실**: spec § "runner workspace ↔ dev workspace 동거 함정 격리" 카논. 4 runner 가 같은 named volume 공유 → atomic file ops race-safe 주장 (PR description) 의 근거 부재 (특히 동시 `pnpm install` 의 `.tmp` rename, Go module download lock).
- **권고**: pnpm-cache 만 runner 별 분리 (`runner{N}-pnpm`) 또는 race-safe 근거 README 보강.

### H-5 [Architecture] Option A (Custom Image) 미선택 정당화 부족
