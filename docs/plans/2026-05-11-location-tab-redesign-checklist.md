# 장소 탭 재설계 체크리스트

## 완료 조건

### 사용자 경험(UX) 완료 조건

- [x] 장소 공개 시점을 라운드가 아니라 게임 플로우 장면 기준으로 저장한다.
- [x] 장소별 출현 장면과 숨김 장면을 선택할 수 있다.
- [x] 숨김 장면 미선택은 마지막 장면까지 공개로 해석한다.
- [x] 장소 탐색 모듈에서 리스트형과 덱형을 선택할 수 있다.
- [x] 덱형은 설정 순서와 랜덤 순서를 선택할 수 있다.
- [x] 장소는 평면 조사 단위로 보이고 하위장소/부모장소 설정은 노출하지 않는다.
- [x] 카드덱형은 별도 슬롯 관리 없이 장소에 추가된 단서를 덱 카드로 보여준다.

### 데이터/계약 완료 조건

- [x] 단서는 한 장소에만 매핑되며, 다른 장소에 배치된 단서는 재배치 전에 중복 추가되지 않는다.
- [x] 서버 저장 검증에서도 같은 단서가 여러 장소 discovery에 중복 매핑되면 거부한다.
- [x] 캐릭터별 장소 접근 제한은 기존 정책을 유지한다.
- [x] DB는 scene 기반 장소 공개 컬럼을 점진 추가하고 기존 round 컬럼은 즉시 삭제하지 않는다.
- [x] 장소 접근 정책에서 scene 가시성 필드를 분리해 책임을 명확히 한다.
- [x] 장소 탐색 모듈은 기존 `location_clue` 런타임 ID와 신규 `location` 에디터 설정을 모두 받을 수 있게 alias를 둔다.
- [x] `theme_locations` SQL 반환 컬럼을 명시해 round 컬럼 잔존 기간의 sqlc drift 위험을 줄인다.

### 품질/검증 완료 조건

- [x] 구버전 라운드 기반 테스트와 fixture를 scene 기반 계약으로 정리한다.
- [x] legacy 장소 단서 설정도 중복 단서 감지에 포함한다.
- [x] 3단계 장소 구조가 신규 UI에서 다시 노출되지 않도록 검증한다.
- [x] 카드덱형 단서 순서 변경은 드래그 앤 드롭으로 검증한다.

## 후속 범위

- [ ] 게임 런타임에서 장소 공개/숨김 scene 조건을 실제 플레이 UI 노출 규칙으로 해석한다.
- [ ] 덱형 조사에서 요청 순서, 유니크 단서 소모, 동시 클릭 충돌 처리를 런타임 엔진에 반영한다.
- [ ] legacy parent location 데이터가 남아 있는 테마를 신규 평면 UI로 안전하게 표시하는 감사 리포트를 추가한다.
