# Location Tab Rework Checklist

## 목표

에디터의 장소 탭을 장면 기반 출현/숨김, 평면 장소 조사, 전역 소모형 단서 획득 규칙으로 정렬해 제작자가 바로 운영 가능한 상태로 만든다.

## 포함 범위

- `locations` 탭 에디터 UI 리빌드
- 장소 노출 정책(scene 기반 출현/숨김)
- 장소별 리스트형/카드덱형 조사 모드
- 장소-단서 매핑과 중복 방지
- 접근 제약 처리
- 점진 마이그레이션

## 제외 범위

- 부모/하위 장소 편집 UI
- 별도 슬롯 추가/삭제/라벨 관리 UI
- 게임 런타임 전체 재작성
- 단서 효과 엔진

## 완료 조건

### 사용자 경험(UX) 완료 조건

- 에디터의 장소 탭에서 장면 기반 출현/숨김 구간을 바로 설정할 수 있음
- 장소가 모두 동등한 조사 단위임이 UI에서 명확히 보임
- 맵은 조사 계층이 아니라 장소 묶음 컨테이너로 보임
- 리스트형/카드덱형 차이가 단서 배치 방식으로 이해됨
- 카드덱형에서 단서를 추가하면 그 단서가 곧 덱 카드가 됨

### 데이터/계약 완료 조건

- 장소와 단서 관계가 UI에서 단일 출처로 관리됨
- 출현/숨김 씬 구간으로 노출이 결정됨
- 단서 중복 매핑이 막힘
- 기존 parent/round 기반 데이터는 점진 호환됨
- 별도 slot 엔티티 없이 discovery 순서가 카드덱 순서로 저장됨

### 품질/검증 완료 조건

- 동시 조사 시 요청순 선점 및 소모성 규칙이 후속 런타임 요구사항으로 추적됨
- 정규화/마이그레이션 후 기존 데이터 읽기 호환성이 유지됨
- 중복 단서/잘못된 legacy 트리 구조 리포트가 가능함
- UI 검증에서 장소 트리 3단계 구조가 다시 노출되지 않음
