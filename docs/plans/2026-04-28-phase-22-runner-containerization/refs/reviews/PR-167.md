## HIGH

### H-SEC-1 [e2e-stubbed.yml:115] pkill cross-kill (parallel shards)

`pkill -f /tmp/mmp-server` 는 4 shard (chromium/firefox × 1/2) 가 같은 self-hosted host (sabyun) 에서 병렬 실행될 때 **다른 shard 의 사용 중인 mmp-server 를 강제 종료** 가능. `concurrency.cancel-in-progress: true` 는 같은 ref 의 이전 run 만 취소하고, 같은 run 내부 4-matrix shard 간 cross-kill 은 막지 못함. shard A startup 직전 shard B 가 health-check 통과해 테스트 실행 중이면 A 의 pkill 이 B 의 server 까지 죽임.

PR description 의 "host runner 단일 사용 보장" 가정은 4 shard matrix 와 충돌. **권고**: `pgrep -u $USER /tmp/mmp-server` 로 user 자기 프로세스만 확인 후 (없으면 skip) 진행, 또는 PID file 기반 cleanup.
