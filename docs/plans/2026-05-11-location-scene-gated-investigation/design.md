# Location Tab Rework — Scene-gated Investigation Design

## 원인

현재 장소 탭은 기존 구현의 유산 때문에 장소 자체, 지도, 단서 노출 흐름이 섞여 있어 장면 기반 탐색 규칙(출현/숨김 씬), 하위 장소(서랍·액자) 단위 조사, 단서 소모형 동시 선점 처리 규칙이 일관되게 드러나지 않습니다.

요구사항은 다음 6가지로 정리됩니다.

- 장소는 게임 장면 단위로 `출현씬`~`숨김씬` 구간에서만 노출
- 하위 장소만 실 조사 단위, 상위 장소는 컨테이너 역할
- 장소 단위로 `리스트형/덱형` 탐색 모드 선택
- 덱형은 다중 슬롯 지원, 슬롯 규칙은 장소 단위로 통일
- 캐릭터별 접근 불가 설정 가능
- 단서는 유니크 소모형(전역 소멸)

## 결과(목표 상태)

에디터에서 장소 탭을 통해 다음이 모두 가능한 상태를 목표로 합니다.

- 장소 계층(부모/자식) 구성
- 씬 기반 노출/숨김 구간 설정(기본 숨김씬 = 마지막 씬)
- 탐색 모드·순번 규칙·슬롯 수·접근 제약을 장소 단위로 설정
- 단서 할당을 슬롯 기반으로 정규화하고 중복 매핑을 막음
- 조건 미충족 장소는 에디터 목록에서 숨김
- 점진 마이그레이션으로 기존 데이터 정합 보전

## 권장 구조

### 1) 데이터 모델 (정규화)

`location_node` 중심으로 구성합니다.

- `location_node`
  - `id`
  - `game_id`
  - `parent_location_id` (nullable)
  - `name`
  - `is_root` / `is_investigable`
  - `appearance_scene_id`
  - `disappearance_scene_id` (기본: 마지막 씬)
  - `investigation_mode` (`LIST` | `DECK`)
  - `draw_order` (`ORDERED` | `RANDOM`)
  - `deck_slots` (정수, 최소 1)
  - `forbidden_character_ids` (선택)

- `location_slot`
  - `id`
  - `location_id`
  - `slot_index`
  - `clue_id`
  - `slot_order`

제약:
- 하위 장소는 `is_investigable=true`, 상위 장소는 `false`
- 단서 중복 매핑 금지: `clue_id`는 하나의 슬롯에만 존재
- 부모 비활성/사라짐은 하위 노출에 즉시 반영

### 2) 노출 규칙

- 기본 규칙: `current_scene >= appearance_scene_id` and `current_scene <= disappearance_scene_id`
- 기본 숨김씬: `마지막 씬`
- 기본값은 제작자가 즉시 시작할 수 있게 UI 기본 선택값으로 세팅
- `disappearance_scene_id` 이전에 사라진 장소는 재출현 없음
- 부모 사라짐 시 하위는 함께 사라짐 (연산에서 자동 폐기)

### 3) 탐색 모드

- **리스트형**: 장소 단서 목록을 표시해 순차/랜덤 규칙 적용
- **덱형**: 동일 장소에 슬롯 여러 개 가능(`deck_slots`)
  - 단일 장소당 draw 규칙은 `ORDERED|RANDOM` 1개만 사용
  - 슬롯은 병렬 구성 가능하지만 정렬 규칙은 공유

### 4) 접근 제약

- 장소별 접근 불가 캐릭터를 설정
- 플레이 중/에디터 미리보기에서 접근 불가일 때 문구 고정:
  - `해당 캐릭터는 탐색이 불가능한 장소입니다`

### 5) 단서 소모/동시 처리(정확성)

- 단서 1개는 전역 1회 획득(누군가 획득 시 즉시 소모)
- 동시에 여러 요청 발생 시 **요청순(서버 처리 순서)**으로 선점
- 획득할 수단이 없으면 `no clues available` 메시지 반환
- UI는 선점 결과를 즉시 반영

### 6) 마이그레이션 전략

- 기존 저장값은 읽기 호환
- 신규 정규화 엔티티로 점진 백필
- 중복 단서 매핑/부모-자식 규칙 위반 데이터는 마이그레이션 리포트로 표시

## 에디터 화면 제안

- 장소 트리 패널: 장소/하위 장소 관계 가시화
- 장소 상세 패널: `name`, `appearance_scene`, `disappearance_scene`, `mode`, `order`, `slots`, `restricted characters`
- 슬롯 패널: 슬롯별 단서 드롭존 및 정렬값
- 조건 미충족/미표시 상태: 씬 범위 바깥 장소는 목록에서 제외
- 하위 장소만 조사 표시 아이콘/라벨

## 구현 우선순위(권장)

1. 장소 엔티티 저장 모델 정렬 + 마이그레이션
2. 출현/숨김 씬 UI + 기본값 처리
3. 장소 모드/슬롯 UI
4. 단서 중복 방지 + 슬롯 정렬/유효성
5. 접근제한 문구/상태 반영
6. 점검 체크리스트 기반 검증

## 위험요인

- 기존 UI에서 지도 기반 경로와 병행 시 설계가 섞이면, 하위장소가 실조사 가능한지/불가능한지 혼동될 수 있음
- 동시 클릭 선점 처리는 클라이언트가 아닌 서버 기준으로 처리하지 않으면 duplicate 획득 버그 가능
