🌍 실제 도구는 어떻게?
📚 Articy:draft (게임 시나리오)
"Entity Database + Template" — entity 종류(Character/Item/Location)와 그 위에 붙이는 Feature 를 분리. Template editor 좌측에 features 리스트, 우측에 entity 폼.
📁 Entities ▾
  👤 Characters (12)
  📍 Locations (8)
  🎒 Items (24)
📁 Flow ▾
📁 Templates (어떤 feature 를 attach)
≈ 우리 B+C 와 거의 동일
🏰 RPG Maker MV/MZ
좌측 트리에 entity 종류 16개 (Map / Actor / Class / Skills / Items / Weapons / Armors / Enemies / Troops / States / Animations ...). 완전 entity-first 극단. 그룹 헤더 X, 평면 리스트.
≈ B 옵션 극단 — 우리는 21 모듈이라 평면 리스트면 너무 김
🌐 World Anvil (TTRPG 세계 빌더)
좌측 트리 = Articles, 사용자가 직접 카테고리 만들어 drag & drop 그룹화. "Settlements / Characters / Magic Systems" 등을 사용자가 정의. 그 위에 "Article Templates" 가 feature 처럼 붙음.
≈ C 의 그룹 헤더 + 사용자 커스터마이즈 가능 (우리는 fixed 3 그룹이라 더 가벼움)
📝 Notion / Linear
사이드바에 uppercase 그룹 헤더 (WORKSPACE / PRIVATE / SHARED) — 클릭 가능 항목 아니고 시각적 라벨. 우리 1️⃣ 2️⃣ 3️⃣ 헤더와 같은 패턴.
≈ C 의 라벨 패턴 차용 근거
🎮 Unity / Godot Inspector
좌측 = Hierarchy(=entity 트리), 우측 = Inspector(=entity 폼 + 동적 component 섹션). 우리 페이지 B 모델의 원형.

왼쪽 mockup 의 사이드바 구조 + 컬럼 동적 변화 + 페이지 B(단서 1개 편집) 흐름이 우리 안. 오른쪽은 같은 문제를 푼 실제 도구들의 패턴.
