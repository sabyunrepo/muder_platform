# 07. 결말 entity + 분기 시스템

> 결정: D-11 (두 탭, 점수 배수 X), D-12 (질문 + 답 매트릭스), D-13 (공통 결말), D-14 (점수 별 차원), D-15 (점수 가시성), D-18 (voting 자동 컬럼).
> 대표 mockup: [q15](../mockups/q15-ending-branch-system.html) → [q19](../mockups/q19-ending-v5-final.html) (v1→v5 진화)

## 결정 사항

결말 페이지 = **두 탭**.

### 탭 1: 결말 목록 (CRUD)

**베이스 필드**:

| 필드 | 타입 | 비고 |
|------|-----|------|
| 라벨 | string | 예: "TRUTH" / "자비" / "정의" |
| 색상/아이콘 | string | 예: "🟢" / "💚" |
| ~~점수 배수~~ | ❌ 제거 | 모든 결말 점수 = 1.0 고정 |

**📜 결말 컨텐츠** (Markdown) — 게임 끝에 모두에게 공개되는 "사건의 전말" 텍스트.

CRUD UX = 다른 entity 와 동일 (좌측 리스트 + "+ 추가" / 우클릭 / 드래그).

### 탭 2: 결말 분기 시스템 (질문 + 매트릭스)

**❓ 게임 끝 질문 (제작자 동적 정의)**:

| 필드 | 비고 |
|------|------|
| 질문 텍스트 | "당신은 범인을 용서하겠습니까?" |
| 종류 | **단일 선택** / **다중 선택** (2 종만) |
| 보기 | 문자열 리스트 (수동 입력 — 자동 채움 X) |
| 응답 캐릭터 | 모두 / 일부 / 1명 (질문마다 선택) |
| **영향 종류** | 🎭 결말 분기 OR 💯 점수 가감 (택1) |

**🎭 결말 분기 질문**: 답이 매트릭스에 입력됨. 모든 응답자 답 다수결로 종합.

**💯 점수 가감 질문**: 보기마다 점수 (+/-/0) 입력. 응답자 각자 점수 누적. **플레이어 답할 때는 점수 ❌ 숨김** (스포일러 방지). 제작자 설정 화면에서만 보임.

### 매트릭스 (답 자체 매칭)

| 컬럼 종류 | 출처 |
|---------|-----|
| 🗳️ **모듈 결과 컬럼 (자동)** | voting / accusation 등 모듈 ON 시 자동 추가. "이영희 검거" 처럼 답 직접 사용. |
| ❓ **질문 컬럼** | 제작자 정의 결말 분기 질문 |
| **`*` 와일드카드** | 어떤 답이든 OK |

매트릭스 예시:
```text
| 🗳️ voting    | Q1 용서? | Q2 물건       | Q5 일기  | → 결말 |
| 이영희 검거    | *      | *            | 읽었다    | TRUTH |
| 이영희 검거    | 예     | *            | *        | 자비  |
| 이영희 검거    | 아니오  | *            | *        | 정의  |
| *           | *      | 현금+반지     | *        | 탐욕  |
| 박/김/최 검거  | *      | *            | *        | 오판  |
| (기본)       |        |              |          | 미해결 |
```

위에서 아래 우선순위. 답 자체 매칭 (정답/오답 X).

### 게임 끝 화면 (3 부분)

1. **공통 결말** (모두 같은 엔딩 봄):
   ```text
   공통 결말: 💚 자비
   🗳️ 검거: 이영희 (다수결)
   ❓ Q1 용서? = 예 (다수결)

   📜 [Markdown 컨텐츠]
   ```

2. **플레이어별 응답 + 점수 breakdown** (랭킹 X):
   ```text
   | 캐릭터  | Q1 | Q2       | Q3 동기      | Q4 발견 | Q5    | 💯  |
   | 김철수  | 예 | 지갑+반지 | 협박(+5)    | R2(+3) | 읽었다 | +8 |
   | 박민수  | 예 | 현금다발  | 협박(+5)    | 몰랐다  | —     | +5 |
   | 이영희  | —  | 지갑     | 치정(-2)    | R1(+5) | —     | +3 |
   | 최지영  | —  | 반지     | 모르겠다(0) | R3(+1) | —     | +1 |
   | 정대호  | — (희생자)                                    | 0  |
   ```

   - 결말 분기 질문 (Q1·Q2·Q5) = 답만 표시
   - 점수 가감 질문 (Q3·Q4) = 답 + 점수
   - "—" = 응답 안 한 캐릭터 (응답 셀렉트 제외 또는 희생자)

## 배경·근거

### 점수 배수 제거 (사용자 명시)

이전 v1 mockup 에 결말마다 x1.5 / x1.0 / x0.5 등 차등. 사용자 결정: **모든 결말 점수 = 1.0**. 결말 차이 = 라벨·색상·컨텐츠만. 점수는 별 차원.

### 정답 컨셉 X (D-12)

이전 v1 = "정답/오답 매트릭스" (예: 이영희 = 정답). 사용자 발화: "질문이 무조건 정답이 있는 게 아니라 — 당신은 누구를 용서하겠습니까? 예/아니오, 또는 돈이 될만한 물건을 몇 개 찾았는지? 1,2,3,4 이런식". 

→ **답 자체 매칭** 채택. 정답 표시는 선택 (있어도 OK, 없어도 OK).

### 응답 캐릭터 셀렉트 (사용자 추가)

질문마다 어느 캐릭터에게 답시킬지 결정. 모두 강제 X. 예:
- Q1 "용서?" → 김철수, 박민수만 (희생자 친구들)
- Q3 "일기 정독?" → 김철수만 (사이드 미션)
- Q2 "물건?" → 모두

### voting 결과 = 자동 컬럼 (D-18)

voting 모듈 결과 (범인 검거 결과) 는 별도 질문 만들 필요 없이 매트릭스에 자동 컬럼으로 추가. 사용자 발화: "범인 검거여부는 범인 투표 모듈에서 나온 결과를 바탕으로". 다른 모듈 (accusation 등) 결과도 확장 가능.

### 공통 결말 vs 개인별 (D-13)

이전 v4 = 개인별 결말 (각 플레이어가 자기 답으로 자기 결말). 사용자 결정 변경: "모든 캐릭터 응답을 종합해서 결말을 정할거야". → **공통 결말** (다수결 종합 1개 결과).

머더미스터리 분위기 — 결말 = "비밀이 풀리는 순간 모두 공유". 점수만 개인 차이.

### 점수 가시성 (D-15)

플레이어 답할 때 점수 표시 ❌ — 점수 노리고 답 결정하는 거 방지 (스포일러). 제작자 설정 화면에선 점수 보임. 게임 끝에서 점수 공개.

### 랭킹 제거

v4 = 1위/2위/3위 표시. 사용자 결정: "랭킹은 필요 없고". 단순 응답+점수 표만.

## UX 디테일 — 3 화면

### 화면 1: 제작자 설정 (에디터)

```text
🎯 결말 페이지 → 탭 2: 결말 분기

❓ 질문 정의

[🔵 결말 분기]
Q1. 당신은 범인을 용서하겠습니까?
   종류: 단일 선택
   응답: 김철수, 박민수
   보기: [예] [아니오]

[🟡 점수 가감]
Q3. 정대호의 정확한 살해 동기는?
   종류: 단일 선택
   응답: 모두 5명
   보기 + 점수:
     - 협박  +5
     - 치정  -2
     - 금전  -2
     - 모르겠다  0

⚙️ 응답 종합 = 다수결 (모든 응답자 답 종합)

🗳️ voting 결과 = 매트릭스 자동 컬럼

🎯 답 조합 → 결말 매핑 매트릭스 (위 표)
```

### 화면 2: 플레이어 답 (게임 중)

```text
┌─────────────────────────────┐
│ 정대호의 정확한 살해 동기는?  │
│ 하나만 선택 · 김철수님       │
│                             │
│ [ 협박 ]                    │
│ [ 치정 ✓ ]    ← 선택        │
│ [ 금전 ]                    │
│ [ 모르겠다 ]                │
│                             │
│ ⚠️ 점수·정답 표시 없음       │
└─────────────────────────────┘
```

점수 비가시. 보기만 단순 클릭.

### 화면 3: 게임 끝

```text
공통 결말: 💚 자비
🗳️ 검거: 이영희 (다수결) | Q1 용서? = 예 (다수결)
📜 [Markdown 컨텐츠]

📊 플레이어별 응답 & 점수 (랭킹 X)
[표 — 위 결정 사항 참조]
```

## 현재 코드 차용

| 영역 | 기존 |
|------|-----|
| 결말 entity CRUD | 신설 — 현재 v3 결말은 Flow 노드 (`EndingNodePanel`) 형식. 별도 entity 페이지 신설 |
| 결말 컨텐츠 Markdown | v2 `contentService.getEnding('TRUTH')` 모델 차용. v3 `theme_contents` 컬럼 보존 |
| 분기 시스템 | 신설 — 백엔드 ConfigSchema 신설 (§O-04) |
| voting 결과 노출 | 기존 voting 모듈 결과를 분기 시스템이 구독하는 wire-up 신설 |

## 미해결 디테일

- **다중 선택 다수결 룰** (§O-08) — "각 보기별 과반?" vs "가장 많이 받은 N개?" → writing-plans 단계 정밀화
- **백엔드 ConfigSchema 신설** (§O-04) — 매트릭스 / 질문 / 점수 시스템 데이터 모델
- **점수 시스템 모듈** (§O-05) — `scoring` 신설 vs 기존 모듈 확장
- **게임 중 다른 사람 답 노출** — 토론 활기 vs 스포일러. 현재 결정 = 게임 끝까지 비공개. 추후 옵션 고려.
- **결말 노드 → 결말 entity 연결** — Flow 다이어그램 결말 노드와 결말 entity 1:1 매핑 (writing-plans).
